# План интеграции с Figma API для автоматических ресайзов

## Обзор задачи

Создать функционал, который:
1. Принимает ссылку на макет в Figma
2. Автоматически парсит структуру макета
3. Генерирует ресайзы для всех размеров из `sizes-config.json`
4. Экспортирует результаты в удобном формате

## Архитектура решения

### 1. Модуль работы с Figma API (`src/utils/figmaApi.js`)

**Основные функции:**
- `extractFileKey(url)` - извлечение ключа файла из URL Figma
- `extractNodeId(url)` - извлечение ID ноды (frame/component) из URL
- `fetchFigmaFile(fileKey, token)` - получение структуры файла через API
- `fetchFigmaNode(fileKey, nodeId, token)` - получение конкретной ноды
- `fetchFigmaImages(fileKey, nodeIds, token, scale)` - получение изображений для нод
- `parseFigmaStructure(fileData, nodeId)` - парсинг структуры макета

**Требования:**
- Нужен Personal Access Token от Figma (пользователь вводит в настройках)
- API имеет rate limits (200 запросов в минуту)

### 2. Парсер структуры макета (`src/utils/figmaParser.js`)

**Функции:**
- `identifyElements(figmaNode)` - идентификация элементов по именам слоев
- `extractTextContent(node)` - извлечение текстового контента
- `extractImages(node)` - извлечение изображений
- `extractStyles(node)` - извлечение стилей (цвета, шрифты, размеры)
- `calculateLayout(node)` - расчет позиций и размеров элементов

**Система именования слоев (рекомендуемая, но не обязательная):**

Для более точного парсинга можно использовать префиксы в названиях слоев:

```
[logo] - Логотип
[title] - Заголовок
[subtitle] - Подзаголовок
[legal] - Лигал
[kv] - Визуал (key visual)
[bg] - Фон
[age] - Возрастное ограничение
[partner-logo] - Логотип партнера
```

Пример структуры в Figma:
```
Frame: Макет 1080x1080
  ├─ [bg] Фон
  ├─ [kv] Визуал
  ├─ [logo] Логотип
  ├─ [title] Заголовок
  ├─ [subtitle] Подзаголовок
  ├─ [legal] Лигал
  └─ [age] 18+
```

**Без специальных имен:**
- Система будет пытаться определить элементы автоматически:
  - Логотип: ищет слои с "logo" в названии или SVG компоненты
  - Текст: определяет по типу TEXT
  - Изображения: определяет по типу IMAGE или компонентам с изображениями
  - Фон: самый нижний слой или слой с "background"/"bg" в названии

### 3. Генератор ресайзов (`src/utils/figmaResizer.js`)

**Функции:**
- `calculateScale(originalSize, targetSize)` - расчет масштаба
- `resizeElement(element, scale)` - масштабирование элемента
- `adjustTextSize(textNode, scale)` - корректировка размера текста
- `generateResizedLayout(originalLayout, targetSize)` - генерация ресайзнутого макета
- `exportResizedFrames(fileKey, nodeId, sizes, token)` - экспорт всех размеров

**Логика ресайза:**
1. Определяем исходный размер макета из Figma
2. Для каждого размера из `sizes-config.json`:
   - Рассчитываем масштаб (может быть разный для width и height)
   - Применяем масштаб к позициям и размерам элементов
   - Корректируем размеры шрифтов пропорционально
   - Генерируем изображение через Figma API

### 4. UI компонент (`src/ui/components/figmaImporter.js`)

**Элементы интерфейса:**
- Поле ввода для URL макета Figma
- Поле ввода для Personal Access Token (с возможностью сохранения)
- Кнопка "Импортировать и ресайзить"
- Прогресс-бар для отслеживания процесса
- Предпросмотр исходного макета
- Список размеров для ресайза (чекбоксы)
- Кнопка "Экспортировать все размеры"

**Интеграция в существующий UI:**
- Добавить новую вкладку "Figma" в панель табов
- Или добавить кнопку в секцию экспорта

### 5. Экспортер Figma макетов (`src/utils/figmaExporter.js`)

**Функции:**
- `exportFigmaLayouts(fileKey, nodeId, sizes, token, format)` - экспорт всех размеров
- `downloadFigmaExport(url, filename)` - скачивание экспортированного изображения
- `createZipFromFigmaExports(exports, format)` - создание ZIP архива

**Процесс экспорта:**
1. Для каждого размера:
   - Создаем временный frame в Figma с нужным размером (или используем API для ресайза)
   - Экспортируем изображение через Figma API (PNG/JPG)
   - Сохраняем в память
2. Собираем все изображения в ZIP архив
3. Предлагаем скачать

## Технические детали

### Figma API Endpoints

1. **GET /v1/files/:key** - получение структуры файла
2. **GET /v1/files/:key/nodes?ids=:ids** - получение конкретных нод
3. **GET /v1/images/:key?ids=:ids&format=png&scale=:scale** - экспорт изображений

### Ограничения API

- Rate limit: 200 запросов в минуту
- Максимальный размер экспорта: зависит от плана Figma
- Форматы экспорта: PNG, JPG, SVG, PDF

### Обработка ошибок

- Невалидный URL
- Недоступный файл (приватный)
- Неверный токен
- Превышение rate limit
- Ошибки сети
- Недостаточно прав на файл

## Пример использования

```javascript
// 1. Пользователь вводит URL
const figmaUrl = "https://www.figma.com/file/abc123/MyDesign?node-id=1:2";

// 2. Система извлекает fileKey и nodeId
const { fileKey, nodeId } = extractFigmaIds(figmaUrl);

// 3. Получаем структуру макета
const fileData = await fetchFigmaFile(fileKey, token);
const nodeData = await fetchFigmaNode(fileKey, nodeId, token);

// 4. Парсим структуру
const layout = parseFigmaStructure(fileData, nodeId);

// 5. Получаем все размеры из конфига
const sizes = getCheckedSizes(); // из sizes-config.json

// 6. Для каждого размера генерируем ресайз
for (const size of sizes) {
  const resizedLayout = generateResizedLayout(layout, size);
  const imageUrl = await exportFigmaImage(fileKey, nodeId, size, token);
  // Сохраняем или добавляем в ZIP
}
```

## Безопасность

- Токен хранится в localStorage с шифрованием
- Токен не отправляется на сторонние серверы
- Все запросы идут напрямую к Figma API
- Предупреждение пользователя о необходимости Personal Access Token
