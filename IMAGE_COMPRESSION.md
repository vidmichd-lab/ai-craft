# Автоматическое сжатие изображений

Скрипт деплоя автоматически сжимает изображения перед загрузкой в S3, что значительно уменьшает размер файлов и ускоряет загрузку сайта.

## Как это работает

1. **Перед деплоем** скрипт `deploy.sh` автоматически запускает `compress_images.py`
2. **Скрипт сжатия** находит все изображения (PNG, JPG, JPEG, WebP, HEIC) в проекте
3. **Сжимает изображения** с оптимальными настройками:
   - Качество: 85% (баланс между размером и качеством)
   - Минимальный размер для сжатия: 20 KB (маленькие файлы пропускаются)
   - PNG: максимальное сжатие без потери качества
   - JPEG: прогрессивное сжатие с оптимизацией
   - WebP: современный формат с лучшим сжатием

## Настройки

### Изменение качества сжатия

В файле `deploy.sh` найдите строку:
```bash
python3 compress_images.py --min-size 20 --quality 85
```

Измените параметры:
- `--quality 85` - качество сжатия (1-100, выше = лучше качество, больше размер)
- `--min-size 20` - минимальный размер файла для сжатия в KB

### Ручной запуск сжатия

Вы можете запустить сжатие вручную:

```bash
# Сжать все изображения в текущей директории
python3 compress_images.py

# Сжать с другими настройками
python3 compress_images.py --quality 90 --min-size 50

# Сжать и ограничить максимальный размер
python3 compress_images.py --max-dimension 1920 --quality 85

# Сохранить сжатые изображения в другую папку
python3 compress_images.py --output compressed/
```

### Параметры командной строки

- `--quality, -q` - качество сжатия (1-100, по умолчанию: 85)
- `--min-size` - минимальный размер файла для сжатия в KB (по умолчанию: 50)
- `--max-dimension, -m` - максимальный размер (ширина или высота в пикселях)
- `--output, -o` - директория для сохранения (по умолчанию: перезаписать оригиналы)

## Что сжимается

- ✅ PNG файлы (оптимизация без потери качества)
- ✅ JPEG/JPG файлы (сжатие с настраиваемым качеством)
- ✅ WebP файлы (если есть)
- ✅ HEIC/HEIF файлы (конвертация в JPEG)
- ❌ SVG файлы (не сжимаются, уже оптимизированы)
- ❌ Маленькие файлы (< 20 KB по умолчанию)

## Установка зависимостей

Скрипт автоматически установит необходимые библиотеки при первом запуске:
- `Pillow` - для работы с изображениями
- `pillow-heif` - для поддержки HEIC/HEIF форматов

Если установка не удалась, установите вручную:

```bash
pip3 install Pillow pillow-heif
```

## Результаты

После сжатия вы увидите:
- Список обработанных файлов
- Размер до и после сжатия
- Процент экономии места
- Общую статистику

Пример вывода:
```
✓ assets/3d/icons/01.png: 245.3 KB → 89.2 KB (63.6% меньше)
✓ assets/3d/icons/02.png: 189.7 KB → 67.1 KB (64.6% меньше)

✅ Сжато файлов: 48
📦 Экономия: 12.3 MB (58.2%)
⏭️  Пропущено: 5
```

## Важные замечания

1. **Оригиналы сохраняются**: По умолчанию скрипт перезаписывает оригинальные файлы. Если хотите сохранить оригиналы, используйте `--output` для указания папки для сжатых версий.

2. **Git**: Сжатые файлы будут закоммичены в Git, если вы делаете commit после деплоя. Это нормально - сжатые версии и есть те, которые должны быть в репозитории.

3. **Производительность**: Сжатие может занять некоторое время при первом запуске или при большом количестве изображений. Последующие запуски будут быстрее, так как сжимаются только изменённые файлы.

4. **Качество**: Качество 85% - это хороший баланс. Для фотографий можно использовать 90%, для иконок и графики - 80%.
