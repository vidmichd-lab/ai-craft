# Lighthouse Audit - Инструкция

## Быстрый способ (Chrome DevTools) - Рекомендуется

1. **Убедитесь, что сервер запущен:**
   ```bash
   python3 start_server.py
   ```

2. **Откройте приложение в Chrome:**
   - Перейдите на http://localhost:8000

3. **Откройте DevTools:**
   - Нажмите `F12` или `Cmd+Option+I` (Mac) / `Ctrl+Shift+I` (Windows/Linux)
   - Или правый клик → "Inspect"

4. **Перейдите на вкладку Lighthouse:**
   - Если вкладки нет, нажмите `>>` и выберите "Lighthouse"

5. **Настройте аудит:**
   - Выберите категории:
     - ✅ Performance (Производительность)
     - ✅ Accessibility (Доступность)
     - ✅ Best Practices (Лучшие практики)
     - ✅ SEO (Поисковая оптимизация)
   - Выберите режим: "Navigation" (для полной загрузки страницы)

6. **Запустите аудит:**
   - Нажмите кнопку "Analyze page load"
   - Дождитесь завершения (обычно 30-60 секунд)

7. **Просмотрите результаты:**
   - Lighthouse покажет оценки по каждой категории (0-100)
   - Нажмите на категорию для детального просмотра
   - Исправьте проблемы, отмеченные красным/желтым

## Автоматический способ (CLI)

### Установка Lighthouse CLI

```bash
npm install -g lighthouse
```

### Запуск аудита

```bash
# Используйте готовый скрипт
./run_lighthouse.sh

# Или вручную
lighthouse http://localhost:8000 \
  --output=html,json \
  --output-path=./lighthouse-report \
  --only-categories=performance,accessibility,best-practices,seo
```

### Просмотр результатов

Откройте `lighthouse-report.html` в браузере для интерактивного отчета.

## Интерпретация результатов

### Performance (Производительность)
- **90-100**: Отлично ⭐
- **50-89**: Хорошо, есть что улучшить
- **0-49**: Требуется оптимизация

**Основные метрики:**
- **FCP (First Contentful Paint)**: Время до первого контента
- **LCP (Largest Contentful Paint)**: Время до самого большого элемента
- **TBT (Total Blocking Time)**: Время блокировки основного потока
- **CLS (Cumulative Layout Shift)**: Стабильность макета

### Accessibility (Доступность)
- **90-100**: Отлично ⭐
- **50-89**: Хорошо, есть что улучшить
- **0-49**: Требуется улучшение

**Основные проблемы:**
- Отсутствие ARIA labels
- Низкий контраст текста
- Отсутствие alt-атрибутов у изображений
- Проблемы с навигацией с клавиатуры

### Best Practices (Лучшие практики)
- Использование HTTPS
- Отсутствие устаревших API
- Безопасность (console errors, mixed content)

### SEO (Поисковая оптимизация)
- Мета-теги
- Структурированные данные
- Мобильная оптимизация

## Типичные проблемы и решения

### 1. Производительность
- **Большие изображения**: Используйте WebP, оптимизируйте размеры
- **Неиспользуемый JavaScript**: Code splitting, lazy loading
- **Блокирующие ресурсы**: Preload критических ресурсов

### 2. Доступность
- **Отсутствие ARIA labels**: Добавьте `aria-label` к интерактивным элементам
- **Низкий контраст**: Улучшите цветовые контрасты (минимум 4.5:1)
- **Отсутствие alt-атрибутов**: Добавьте описательные alt к изображениям

### 3. Best Practices
- **Console errors**: Исправьте ошибки в консоли
- **HTTPS**: Используйте HTTPS в production

## Рекомендации после аудита

1. **Исправьте критичные проблемы** (красные)
2. **Улучшите предупреждения** (желтые)
3. **Проверьте снова** после исправлений
4. **Документируйте изменения** в этом файле

## Периодичность проверки

- **Перед релизом**: Обязательно
- **После больших изменений**: Рекомендуется
- **Ежемесячно**: Для мониторинга деградации
